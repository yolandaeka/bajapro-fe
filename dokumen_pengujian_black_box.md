# DOKUMEN SKENARIO PENGUJIAN BLACK BOX TESTING - APLIKASI WEB BAJAPRO (LMS JAVA)

Dokumen ini berisi skenario pengujian fungsional (Black Box Testing) untuk aplikasi web pembelajaran Java berbasis Learning Management System (LMS) **BAJAPRO**. Pengujian difokuskan pada fungsionalitas utama sistem yang terbagi berdasarkan tiga peran (role) pengguna: **Admin**, **Pengajar (Teacher)**, dan **Pelajar (Student)**.

Skenario pengujian mencakup kasus pengujian positif (input valid) dan negatif (input tidak valid/edge case) untuk memastikan keandalan, keamanan, dan fungsionalitas sistem berjalan sesuai dengan spesifikasi kebutuhan aplikasi.

## CATATAN UNTUK PENGUJI (TESTER)
1. **Hasil Aktual** harus diisi sesuai dengan apa yang terjadi di layar saat langkah uji dijalankan (misal: "Sesuai dengan ekspektasi", atau jika gagal: "Muncul error 500 saat klik simpan").
2. **Status** diisi dengan:
   - **PASS** jika *Hasil Aktual* sama dengan *Hasil yang Diharapkan*.
   - **FAIL** jika terdapat ketidaksesuaian atau error/bug yang menghalangi tercapainya *Hasil yang Diharapkan*.
   - **BLOCK** jika pengujian tidak bisa dilakukan karena fitur pendukung di langkah sebelumnya mengalami kerusakan.
3. Seluruh pengujian ini disarankan dijalankan di lingkungan *Staging* atau *Development* menggunakan database pengujian terisolasi.
